Плагин для **QGIS 4.0** — набор инструментов для **чистки и построения топологии линейных
сетей** (дороги, реки, трубопроводы, ЖД) с сохранением геометрии и атрибутов.

> ⚠️ Требуется **QGIS 4.0 или новее** (Qt6 / PyQt6). На QGIS 3.x не загрузится.

## Зачем это нужно

Линейные данные из открытых (и не очень) источников — **OpenStreetMap**, оцифровка
CAD/DWG, сканы карт, GPS-треки — рисуются так, чтобы *выглядеть* правильно, а не чтобы
*быть* топологически корректными. Две дороги пересекаются без общего узла, ручей не
дотягивает пары сантиметров до реки, в которую впадает, линия проскакивает за перекрёсток,
одна и та же дорога здесь — длинной линией, а там — десятком обрывков. На экране норм; как
**граф — сломано**.

А почти всё полезное, что делают с линейной сетью, требует именно графа:

- **Маршрутизация / сетевой анализ** — дороги, транспорт, ЖД (pgRouting, GRASS `v.net`, сетевые инструменты QGIS);
- **Гидрология** — связные речные сети, трассировка стока, водосборы;
- **Инженерные сети** — вода, газ, электрика, канализация: связность и трассировка вверх/вниз по потоку;
- **Любые древо- или сетеобразующие системы**, которые нужно построить из сырых линий.

Если исходные данные ещё не нодированы и не связны, эти инструменты **молча дают неверный
ответ** — маршрут не проходит через перекрёсток, «оторванный остров» на деле оказывается
зазором в 30 см.

**Network Topology** превращает сырую геометрию в чистый связный граф: достраивает недотяги,
обрезает перехлёсты, разбивает в каждом пересечении и касании, схлопывает лишние псевдоузлы —
а затем даёт *проверить* результат (связные компоненты, оболочки кластеров, степени узлов) и
убедиться, что сеть действительно связна как единое целое. Он собирает рутинные, муторные
шаги — которые иначе пришлось бы городить из десятка инструментов ядра/GRASS — в один провайдер
QGIS Processing, сохраняя геометрию и атрибуты на всех этапах.

Это **слой предобработки и контроля качества**, который запускают *перед* предметным анализом:
не замена вашему роутеру или гидрологической модели, а то, что делает их результатам можно
доверять.

Инструменты — в **Панели инструментов анализа**, в группе **Network Topology → Топология**.

## Инструменты

### 1. Топологическое разбиение — `network_topology:topologysplit`
Нодирует один слой линий сам против себя: разбивает обе линии в каждом **пересечении** (X) и
там, где **конец** одной линии касается другой (T); по желанию **достраивает висячий конец**
вдоль его направления в пределах **допуска**, пока он не достигнет другой линии.
Выход — одночастные линии от узла до узла.

```bash
qgis_process run "network_topology:topologysplit" \
  --INPUT=roads.gpkg --TOLERANCE=0.5 --OUTPUT=roads_noded.gpkg
```

| Параметр | Значение |
|----------|----------|
| `INPUT` | Слой линий для самонодирования |
| `TOLERANCE` | Макс. зазор (ед. карты) для достройки висячего конца. `0` = без достройки |
| `OUTPUT` | Нодированный одночастный слой линий |

### 2. Разрешение висячих концов — `network_topology:resolvedangles`
Чистит висячие концы без разбиения: **достраивает недотяги** вдоль их направления и
**обрезает перехлёсты** до ближайшего пересечения, в пределах одного допуска. Один объект на
выходе на каждый входной объект.

```bash
qgis_process run "network_topology:resolvedangles" \
  --INPUT=roads.gpkg --TOLERANCE=1.0 \
  --FIX_UNDERSHOOTS=true --FIX_OVERSHOOTS=true --OUTPUT=roads_clean.gpkg
```

| Параметр | Значение |
|----------|----------|
| `TOLERANCE` | Макс. зазор для достройки / хвост для обрезки (ед. карты) |
| `FIX_UNDERSHOOTS` | Достраивать концы, не дошедшие до линии |
| `FIX_OVERSHOOTS` | Обрезать концы, проскочившие за пересечение |

### 3. Схлопывание псевдоузлов — `network_topology:collapsepseudonodes`
Объединяет цепочки линий, сходящихся только в **узлах степени 2**, в единые линии
(перекрёстки и тупики сохраняются). Необязательное **поле группировки** запрещает слияние
через границу значений атрибута.

```bash
qgis_process run "network_topology:collapsepseudonodes" \
  --INPUT=roads_noded.gpkg --GROUP_FIELD=road_class --OUTPUT=roads_merged.gpkg
```

| Параметр | Значение |
|----------|----------|
| `GROUP_FIELD` | (необяз.) сливать только там, где это поле равно |
| `OUTPUT` | Объединённый одночастный слой линий |

### 4. Связные компоненты — `network_topology:connectedcomponents`
Помечает каждую линию **независимой подсетью**, к которой она относится: добавляет `cluster_id`
(по убыванию размера, `1` = крупнейшая сеть) и `cluster_size` (число рёбер). Чистая сеть — одна
компонента; лишние компоненты указывают на разрывы или отдельные объекты.

```bash
qgis_process run "network_topology:connectedcomponents" \
  --INPUT=roads_noded.gpkg --OUTPUT=roads_clusters.gpkg
```

| Параметр | Значение |
|----------|----------|
| `INPUT` | Слой линий (сначала нодируйте для корректной связности) |
| `OUTPUT` | Линии с `cluster_id` + `cluster_size` |

### 5. Оболочки кластеров — `network_topology:clusterhulls`
Один **полигон на каждую связную подсеть**: выпуклая оболочка, ограничивающий прямоугольник
или ориентированный прямоугольник. Каждый несёт `cluster_id`, `edge_count` и `length`.
Помогает увидеть, где лежат несвязанные кластеры («остров» вдали от основной сети).

```bash
qgis_process run "network_topology:clusterhulls" \
  --INPUT=roads_noded.gpkg --METHOD=0 --OUTPUT=clusters.gpkg
```

| Параметр | Значение |
|----------|----------|
| `METHOD` | `0` выпуклая оболочка · `1` ограничивающий прямоугольник · `2` ориентированный прямоугольник |
| `OUTPUT` | По полигону на кластер (`cluster_id`, `edge_count`, `length`) |

### 6. Узлы сети — `network_topology:networknodes`
Одна **точка на узел** с его `degree` (степенью) и типом `node_type`: `dangle` (степень 1),
`pseudo` (степень 2), `junction` (степень 3+). Инструмент контроля качества — находит тупики,
псевдоузлы и настоящие перекрёстки.

```bash
qgis_process run "network_topology:networknodes" \
  --INPUT=roads_noded.gpkg --OUTPUT=nodes.gpkg
```

| Параметр | Значение |
|----------|----------|
| `INPUT` | Слой линий (сначала нодируйте, чтобы концы встречались в узлах) |
| `OUTPUT` | Точки с `node_id`, `degree`, `node_type`, `edge_count` |

> Типовой сценарий чистки: **Разрешение висячих концов** → **Топологическое разбиение** →
> **Схлопывание псевдоузлов**; затем **Связные компоненты** / **Оболочки кластеров** /
> **Узлы сети** — для проверки топологии.

## Установка

### Из готового zip
1. Скачайте `network_topology-<версия>.zip` из
   [последнего релиза](https://github.com/Oksion/network-topology-qgis/releases/latest)
   (или соберите сами через `scripts/package.ps1`).
2. В QGIS: **Модули → Управление и установка модулей → Установить из ZIP**.

### Без установки — из Python-консоли QGIS
Вставьте [console/topology_split_console.py](console/topology_split_console.py) в
**Python-консоль** QGIS (`Ctrl+Alt+P`). Сначала выделите слой линий (или задайте
`INPUT_LAYER`/`TOLERANCE` в блоке CONFIG сверху) — скрипт добавит нодированный
временный слой в проект, установка плагина не нужна.

### Для разработки (копирование в профиль)
```powershell
./scripts/deploy.ps1        # копирует плагин в ваш профиль QGIS 4
```
Затем включите **Network Topology** в менеджере модулей и используйте *Plugin Reloader*,
чтобы подхватывать изменения кода без перезапуска QGIS.

Полная настройка среды — в [docs/development.md](docs/development.md).

## Статус

`0.2.0` — экспериментальный. Геометрия обрабатывается как планарная **2D** (Z/M отбрасываются).
Длинные коллинеарные наложения нодируются только по краям; проход достройки концов —
однопроходный. Проверяйте на своих данных перед использованием в продакшене.
